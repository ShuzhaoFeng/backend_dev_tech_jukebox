TouchTunes Backend Dev Tech Assignment

The project is built on Spring Boot and Gson and runs on port number 8080.
Before starting the project, please make sure that Maven is available for the project (Although I do realize that you shouldn't be worried about it if you are seeing this page...).
The project is also Dockerized, you can choose to run the project either by running Main.java or Dockerfile.
This is a read-only project, meaning that I'll get data directly from the mocked APIs and the database cannot be modified from here.
For detailed information about implementation, please refer to JavaDoc files in the project.
The API supports the following query parameters:
  • /api - returns all available Jukeboxes, scraped from touchtunes/tech-assignment/jukes.
  • /api?id={jukebox_id} - returns Jukeboxes with selected IDs.
  • /api?model={model_name} - returns Jukeboxes of the selected models.
  • /api?settingid={setting_id} - returns Jukeboxes that are available for the selected setting, i.e. it contains all components required by the setting.
  • /api?offset={num_offset} - add selected number of blank lines to the page before the result.
  • /api?limit={num_limit} - define the maximum number of lines for the page. All lines that come after the limit will be cropped.
Note that id and model can take multiple entries, while others can only take up to 1 entry at a time.
